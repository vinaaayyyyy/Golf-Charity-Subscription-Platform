Draw and reward system

The draw mechanics are now explained as a real system, not just decorative content.

Draw types

5-number, 4-number, and 3-number match tiers.

These tiers now appear as concrete frontend logic rather than a vague “win prizes” promise.

Logic modes

Random generation or algorithmic weighted draw.

The admin UI now references both random and frequency-weighted logic as required operational options.

Operational cadence

Monthly execution, simulation before publish, rollover when needed.

The system messaging now calls out monthly cadence, admin publishing, and jackpot carry-forward behavior.

Input rules Users enter only the latest 5 scores, each with a Stableford value between 1 and 45 plus a played date
Retention Only the most recent 5 are retained, newest first, and the oldest score is replaced automatically
Publish flow Admin runs simulations before official publish so the draw can be previewed before results lock
Rollover The 5-match jackpot rolls forward to next month if no winner lands the top tier
